# **채팅 서비스 요구사항 정의 (Requirements)** 

## **주요 기능**

- 로그인
- 회원가입
- 채팅방 개설
- 채팅방 참여
- 메세지 전송 (텍스트/이모지)
- 좋아요
- 답장
- 내 메세지 삭제

## **기능 상세**

- 로그인 : 입력필드(이메일, 비밀번호) 버튼(로그인), 링크(회원가입, '비밀번호 찾기')
- 회원가입 : 입력필드(닉네임, 이메일, 비밀번호, 비밀번호 확인), 버튼(회원가입), 링크('로그인으로 이동')
- 홈페이지 :
  - header : 로고, '채팅방 목록', '마이페이지', 사용자정보(이메일), '로그아웃'
  - '모든 채팅방' 문구,  '채팅방 추가하기' 버튼
    - 채팅방 목록 표시:
        - 채팅방 정보 : 아이콘, 방제목, 방장, 최근 메시지 입력일시
- 채팅방 추가 ('/chat-rooms/new')
  - 링크: '뒤로가기', 표시: '채팅방 추가'
  - 표시: '새로운 채팅방 만들기', 설명: 채팅방 이름을 입력하고 '추가' 버튼을 클릭하세요.
    - 입력 필드:
        - 채팅방 이름
        - 설명: 최대 100자까지 입력이 가능합니다.
    - 버튼: '추가', '취소'
- 채팅방 상세 ('/chat-rooms/:id')
  - 메시지 표시
    - 입력한 메시지, 더보기 메뉴('삭제') , 입력 일시, 좋아요 아이콘을 표시한다.
  - 메시지 입력
    - 메시지를 입력하고 엔터를 치면 메시지를 전달한다.
  - 메시지 답장 : 
    - 표시된 메시지와 같이 표시된 더보기 메뉴를 클릭하면 '답장','좋아요 취소' 서브메뉴가 표시된다.
    - '답장' 을 선택하면, 메시지 입력창에 '[누구]님에게 답장 중 + 메시지 내용을 표시하고 그 아래에 메시지를 입력할 수 있도록 한다.
  - 메시지 삭제
    - 자신이 작성한 메시지의 더보기 메뉴를 선택하면 '삭제' 서브메뉴가 표시된다.
    - 삭제 서브메뉴를 선택하면 본인이 작성한 메시지일 경우, 이를 삭제한 후, '삭제되었습니다' 문구를 표시한다.
- 마이페이지
  - 제목 : 마이페이지, 설명: 내 정보를 확인하고 수정할 수 있습니다.
  - 프로필 정보 : 
    - 설명: 회원 가입시 등록된 정보입니다.
    - 이메일: 예시(example@gmail.com)
    - 가입일: 예시(2025년 11월 22일)
  - 닉네임 수정 :
    - 설명: 채팅방에서 표시될 닉네임을 변경할 수 있습니다.
    - 입력: 새 닉네임
    - 버튼: '변경 사항 저장'
  - 버튼: '로그아웃'

## **주의사항**

- Supabase Auth를 사용할 때 클라이언트 사이드와 서버 사이드의 인증 방식을 꼼꼼히 체크하여 꼬이는 경우가 없도록 해야 한다. 
- 상태로 관리해야하는 것만 상태로 관리된다.
- Flux 아키텍쳐가 반영된다.
- Context를 사용해 비즈니스로직을 중앙화 관리한다.
- 채팅 데이터 연동에 단순 polling으로 구현하세요. [주의] realtime, websocket 을 사용하지 않는다.
